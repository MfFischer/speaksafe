# Contributing to SpeakSafe

Thank you for your interest in contributing to SpeakSafe! This project aims to create a secure, anonymous whistleblowing platform that can make a real difference in fighting corruption worldwide.

## 🌟 Ways to Contribute

### 🔧 Development
- **Frontend Development**: React, TypeScript, Tailwind CSS
- **Smart Contract Development**: Solidity, Hardhat, OpenZeppelin
- **Backend Development**: Node.js, Express, PostgreSQL
- **Zero-Knowledge Proofs**: Circom, snarkjs implementation

### 🔒 Security
- **Security Auditing**: Smart contract and application security
- **Penetration Testing**: Finding vulnerabilities
- **Privacy Review**: Ensuring anonymity protection

### 🌍 Accessibility & Localization
- **Accessibility Testing**: WCAG compliance verification
- **Translation**: Multi-language support
- **Regional Legal Research**: Whistleblower protection laws

### 📚 Documentation
- **Technical Documentation**: API docs, architecture guides
- **User Guides**: How-to documentation for end users
- **Legal Documentation**: Compliance and regulatory guidance

## 🚀 Getting Started

### Prerequisites
- Node.js 18+
- Git
- Basic understanding of React and blockchain concepts

### Development Setup
```bash
# Fork and clone the repository
git clone https://github.com/YOUR_USERNAME/speaksafe.git
cd speaksafe

# Install frontend dependencies
cd frontend
npm install

# Start development server
npm start
```

### Project Structure
```
speaksafe/
├── frontend/          # React frontend application
├── contracts/         # Smart contracts (coming soon)
├── backend/          # Backend API (coming soon)
├── docs/             # Documentation
└── README.md
```

## 📋 Development Guidelines

### Code Style
- Use TypeScript for type safety
- Follow ESLint and Prettier configurations
- Write meaningful commit messages
- Add comments for complex logic

### Security Considerations
- Never commit private keys or sensitive data
- Follow security best practices for Web3 development
- Consider privacy implications of all features
- Test thoroughly before submitting PRs

### Pull Request Process
1. Fork the repository
2. Create a feature branch (`git checkout -b feature/amazing-feature`)
3. Commit your changes (`git commit -m 'Add amazing feature'`)
4. Push to the branch (`git push origin feature/amazing-feature`)
5. Open a Pull Request
